1.OCP:
    什么是OCP原则：
        OCP是软件七大开发原则中最基本的一个原则：开闭原则
        开闭原则：对扩展开放，对修改关闭
    OCP是最基本，最核心的，其他六个原则都是建立在这个原则上的
    OCP开闭原则的核心：
        只要你在扩展系统功能的时候，没有修改之前写好的代码，那么你就是符合OCP原则的
        反之，如果在扩展系统功能的时候修改了之前写好的代码，那么就违背了OCP原则
        因为如果在原来的基础上修改了代码的话，那么原来所有的代码都需要重新进行测试
2.依赖倒置原则：
    内容：面向接口编程，面向抽象编程，不要面向具体编程
    目的：降低程序的耦合度，提高扩展力
    什么叫符合依赖倒置：上不依赖下
    违背：上依赖下
3.控制反转：
    反转的是：
        1、不在程序中采用硬编码的方式创建对象
        2、不在使用硬编码的方式维护对象之间的关系
    控制反转是一种编程思想，是一种新的设计模式，由于比较新，没有被纳入GoF23种设计模式中

    控制反转是思想，依赖注入是这种思想的具体实现
    依赖注入，分为两种方式：
        1.set注入
        2.构造方法注入
    依赖注入中的”依赖“和”注入“分别指的是：
        1.依赖：A对象和B对象的关系
        2.注入：是一种手段，通过这种手段可以让A对象和B对象之间产生关系
        依赖注入：对象A和对象B之间的关系，靠注入的手段来维护，而注入包括：set注入和构造注入
